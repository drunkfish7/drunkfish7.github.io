解锁网易云音乐变灰歌曲——送给男女朋友的音乐礼物

UnblockNeteaseMusic 是一位大大开发的小工具，可以帮助解锁网易云音乐客户端变灰歌曲，主要原理是通过其他音源（如酷我等等）替换变灰歌曲。

使用路由器

UnblockNeteaseMusic 有 OpenWrt 端，参见这里或这里。如果已经在用软路由，可以直接刷已经内置 UnblockNeteaseMusic 的固件，固件可以自己编译或者到恩山论坛 x86 区找其他大大编译好的（小弟用的是这个）。

刷好固件后进入路由器后台，找到「解锁网易云灰色歌曲」启用保存即可，另外还可以选择解锁程序或者音源等等。

启用后，路由器自动分流解锁，大部分设备无需配置即可使用。iOS 设备需要先安装这个证书，之后在设置 > 通用 > 关于本机 > 证书信任设置，信任证书（参见官方教程）。macOS 下载这个证书，参照这里信任证书。如果想在电脑浏览器中使用，可以额外安装 Tampermonkey 插件的 NeteaseMusic UI Unlocker 脚本。

解锁是否成功可以在网易云音乐搜索没有版权的歌曲。如「周杰伦 晴天」，如果可以正常播放说明解锁成功。

在软路由上使用可以兼顾科学上网与解锁音乐，设备也可免过多配置，很方便。

群晖 Docker 搭建

另外还可以在群晖 Docker 上安装。打开群晖 Docker，在「注册表」中搜索「unblockneteasemusic」，

待映像下载完映像后（群晖会有系统通知），在「映像」中找到「UnblockNeteaseMusic」，点击「启动」，选择「高级设置」，

脚本默认使用 8080 端口，可以在「端口设置」中根据需要填写，

在「环境」中输入命令 -s -e https://music.163.com ，其中 -s 参数是限制仅网易云音乐所属域名的请求走代理，-e 参数是设置转发音源的域名，确保 iOS 端可以用，具体参见这里的第 2 点。

设置好之后启动容器。

使用时各端需要在 WiFi 配置中填入 PAC 代理脚本地址 http://<群晖 IP:端口号>/proxy.pac，参见下面：

平台 基础设置
Windows 设置 > 工具 > 自定义代理 (客户端内)
UWP Windows 设置 > 网络和 Internet > 代理
Linux 系统设置 > 网络 > 网络代理
macOS 系统偏好设置 > 网络 > 高级 > 代理
Android WLAN > 修改网络 > 高级选项 > 代理
iOS 无线局域网 > HTTP 代理 > 配置代理

和路由器端一样，iOS 设备使用时需要先安装这个证书，之后在设置 > 通用 > 关于本机 > 证书信任设置，手动信任证书（参见官方教程）。另外，macOS 设置代理无法使用，有大大有用 Proxifier 成功的，参见这里。

可以在群晖中设置好 DDNS 、路由器中设置一个端口转发规则，这样就可以在外面的 WiFi 中使用了。

以上仅限于 WiFi 下使用，蜂窝网络使用及 WiFi 下与科学上网共存，小弟没有研究。另外，这个脚本还有衍生项目，参见这里。

其他几个听音乐的地方

如果不想折腾软路由、群晖，也不愿在服务器或本地电脑上折腾，可以试试以下方法：

YouTube

在 YouTube 上几乎可以找到所有歌曲（除非太过于冷门）的 MV，如果想听某首歌，可以直接到 YouTube 搜索。用 YouTube 听音乐也有限制，想听多首歌需要建好播放清单，电脑端不支持后台快捷键切歌、手机客户端非 YouTube Premium 会员无法后台播放。

Listen 1 浏览器插件

Listen 1 是一个电脑浏览器插件，支持 Chrome、Firefox，可以支持多个音乐平台搜索歌曲，可以做一个电脑端的备选。
